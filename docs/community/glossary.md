# Glossary

## Purpose

Provide canonical definitions for core LifeOS terms so the docs set uses stable language.

## Personal AI Node

The local system boundary for Phase 1. A user-controlled AI operating layer that coordinates reasoning, context, automation, modules, and integrations.

Today in repo: foundation context.

## Life Graph

A connected representation of the user's goals, tasks, projects, people, events, resources, and related context.

Today in repo: current MVP.

## Module

A domain-specific extension that adds specialized knowledge, workflows, or integrations to the Personal AI Node.

Today in repo: current MVP.

## Event Model

The shared language of state changes and signals that lets components coordinate without tight coupling.

Today in repo: current MVP.

## Reasoning

The planning and orchestration capability that interprets user requests and system context into useful actions or recommendations.

Today in repo: foundation context.

## Local First

A design posture where processing, storage, and control default to user-owned infrastructure rather than remote services.

Today in repo: current MVP.

## Agent Mesh

A bounded set of specialist roles inside the Personal AI Node that coordinate through shared context and events.

Today in repo: foundation context.

## Simulation

A local decision-support capability that models possible outcomes using life-graph data, production data, and other relevant inputs.

Today in repo: foundation context.

## Room Awareness

The ability to infer which room a person is in so LifeOS can route media, prompts, and automations contextually.

Today in repo: foundation context.

## Follow-Me Media

A behavior where media playback or audio routing moves with the user's detected location.

Today in repo: foundation context.

## AI Receptionist

A voice and telephony workflow that can answer calls, screen callers, take messages, and schedule callbacks through local orchestration.

Today in repo: foundation context.

## Personal Economic Planning

A module area that evaluates income, production, inventory, time allocation, and market signals to support practical planning decisions.

Today in repo: foundation context.

## LifeOS

The operating context and product umbrella for the current personal operations platform.

Today in repo: current MVP.

## Personal Operations OS

The product framing for a system that helps users capture, triage, plan, execute, and review daily work through an AI-assisted loop.

Today in repo: current MVP.

## daily hero loop

The primary end-user flow that moves from capture through review and is treated as the core reliability target for current delivery.

Today in repo: current MVP.

## event bus

The shared messaging layer used by runtime components and modules to publish and subscribe to lifecycle and workflow events.

Today in repo: current MVP.

## runtime module

A module loaded and executed by the platform runtime to provide domain behavior through contracts and event-driven orchestration.

Today in repo: current MVP.

## orchestrator

A coordination layer that sequences module and system actions under policy and workflow rules.

Today in repo: foundation context.

## mesh

A distributed coordination surface for bounded delegation and agent-style role execution.

Today in repo: foundation context.

## bounded delegation

A delegation posture where responsibilities are explicitly constrained by scope, policy, and recoverable fallbacks.

Today in repo: foundation context.

## local-first

An implementation posture where core user data and control paths remain local by default, with remote dependencies treated as optional.

Today in repo: current MVP.

## current MVP

The actively committed product surface and behavior set that contributors should improve and validate first.

Today in repo: current MVP.

## foundation phase

Architecture and platform work that underpins reliability and expansion without claiming immediate end-user breadth.

Today in repo: foundation context.

## roadmap phase

A named stage in planned delivery sequencing used to communicate what is active now versus what is future-facing.

Today in repo: current MVP.
