# Glossary

## Purpose

Provide canonical definitions for core LifeOS terms so the docs set uses stable language.

## Personal AI Node

The local system boundary for Phase 1. A user-controlled AI operating layer that coordinates reasoning, context, automation, modules, and integrations.

## Life Graph

A connected representation of the user's goals, tasks, projects, people, events, resources, and related context.

## Module

A domain-specific extension that adds specialized knowledge, workflows, or integrations to the Personal AI Node.

## Event Model

The shared language of state changes and signals that lets components coordinate without tight coupling.

## Reasoning

The planning and orchestration capability that interprets user requests and system context into useful actions or recommendations.

## Local First

A design posture where processing, storage, and control default to user-owned infrastructure rather than remote services.

## Agent Mesh

A bounded set of specialist roles inside the Personal AI Node that coordinate through shared context and events.

## Simulation

A local decision-support capability that models possible outcomes using life-graph data, production data, and other relevant inputs.

## Room Awareness

The ability to infer which room a person is in so LifeOS can route media, prompts, and automations contextually.

## Follow-Me Media

A behavior where media playback or audio routing moves with the user's detected location.

## AI Receptionist

A voice and telephony workflow that can answer calls, screen callers, take messages, and schedule callbacks through local orchestration.

## Personal Economic Planning

A module area that evaluates income, production, inventory, time allocation, and market signals to support practical planning decisions.
