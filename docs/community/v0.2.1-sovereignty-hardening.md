# v0.2.1 Sovereignty Hardening and Trust UX

This release focuses on three outcomes:

- stronger fail-closed defaults for security-critical paths
- clearer ownership language across CLI and desktop
- transparent trust reporting surfaces so users can inspect runtime behavior

## Highlights

- Implemented `@lifeos/policy-engine`, `@lifeos/security`, and `@lifeos/observability` foundations.
- Tightened module runtime defaults:
  - manifest required by default
  - strict runtime permission mode by default
  - broader wildcard subscribe scopes limited to trusted system modules
- Added CLI trust commands:
  - `lifeos trust status`
  - `lifeos trust explain <action>`
  - `lifeos trust report --json`
- Added Desktop **Trust Center** screen with:
  - local vs cloud posture summary
  - active module and permission visibility
  - recent trust decision timeline
- Expanded desktop settings with explicit ownership and transparency toggles.

## Behavioral Changes

- Policy enforcement now defaults to enabled (`LIFEOS_POLICY_ENFORCE=true` behavior).
- Module manifest requirement now defaults to enabled.
- Module runtime permission mode now defaults to `strict`.

Compatibility note:

- Legacy manifest-less environments can temporarily opt out with `LIFEOS_ALLOW_LEGACY_MANIFESTLESS=true`.

## Why this matters

LifeOS aims to be welcoming and transparent while preserving user ownership.  
This pass makes the security posture stricter without hiding methods from users.
