# Privacy and Sovereignty

## Purpose

State the ownership and privacy posture that should guide all LifeOS design work.

LifeOS is built on the assumption that the user should control their data, context, and automation logic.

## Core Position

- user data should default to local control
- models and reasoning should prefer local execution where practical
- collaboration should share derived insights rather than raw personal data
- the node should remain understandable and inspectable by its owner
- storage and service boundaries should be designed with encryption and zero-trust assumptions in mind
- remote access should prefer user-controlled VPN paths over exposed public surfaces

## Why It Matters

Privacy is not a feature added later. It is part of the reason to build a Personal AI Node at all.
