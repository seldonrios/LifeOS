# Integrations

## Purpose

Describe the local-first external systems that the Personal AI Node should be able to connect to in Phase 1.

## Likely Integration Categories

- Home Assistant and smart home ecosystems
- ESPHome and Bermuda BLE tracking for room awareness
- calendars and task systems
- wearable and fitness data sources
- sensors, cameras, and environmental telemetry
- local files, notes, and knowledge stores
- PBX and VoIP systems for receptionist flows
- media servers, room speakers, and smart displays

## Integration Rule

Integrations should strengthen the local node rather than pull the system into centralization. External services can be useful, but the node should remain the controlling context where possible.

## Why Integrations Matter

Without integrations, LifeOS is only a concept layer. Integrations are what let reasoning, automation, and modules operate on real context.
