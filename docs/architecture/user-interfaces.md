# User Interfaces

## Purpose

Describe how a Phase 1 user interacts with LifeOS across voice, mobile, dashboards, and terminal surfaces.

## Interface Types

- voice interaction in rooms
- mobile app or mobile-friendly control surface
- dashboard displays for home and production context
- terminal interface for power users and operators

## Dashboard Content

Useful dashboard surfaces include:

- life graph summaries
- production status
- goals and active projects
- alerts and opportunities
- communications or callback queues

## Daily Usage Example

```text
Good morning.
Harvest basil today.
Farmers market demand is high.
Jam session is Saturday.
Recommend practice tonight.
```

The interface layer should make the same underlying system visible in different contexts rather than creating separate product silos for each surface.
