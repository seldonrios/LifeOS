# Phase 1 Use Cases

## Purpose

Show the kinds of real user workflows the Personal AI Node should support first.

## Home Automation

The node can interpret context from local devices and drive useful automations such as notifications, environment control, or room-based responses.

## Room-Aware Assistance

The node can infer where a person is in the home and route context accordingly, such as moving audio, selecting the right speaker, or timing prompts to the room the user is actually in.

## Personal Knowledge

The node can organize goals, projects, notes, events, and tasks into a life graph that supports planning and retrieval.

## Productivity

The node can help coordinate calendars, tasks, priorities, routines, and reminders using local reasoning and event-driven workflows.

## Small Production Systems

The node can support small-scale operations such as hydroponics, hobby farming, 3D printing, or online shop workflows through domain modules and local integrations.

## AI Receptionist and Communications

The node can answer calls, screen callers, take messages, or schedule callbacks using local orchestration around a PBX and voice stack.

## Personal Economic Planning

The node can combine production output, inventory, income streams, time allocation, and market signals to support practical decisions such as what to produce more of or where margins are weak.

## Health and Fitness

The node can aggregate wearable or self-tracked data to support routines, planning, and recommendations while keeping the user in control of the data boundary.

## Social Coordination

The node can track practice, local events, and relationships strongly enough to suggest activities such as a jam session, prep tasks, or follow-up messages.

## Daily Briefing

The node can synthesize the life graph, event streams, production state, health context, and calendar into a useful morning or end-of-day summary.
